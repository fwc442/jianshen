#健身房会员管理系统

##项目介绍
本项目是基于 C 语言开发的健身房会员综合管理系统，采用控制台界面实现，面向健身房日常运营场景设计。系统支持管理员与普通用户双权限模式，集会员信息管理、打卡签到、会员卡充值与查询、数据持久化存储等功能于一体，能够高效完成会员开卡、信息修改、打卡统计、数据保存等核心业务，满足小型健身场馆的信息化管理需求。

##系统架构
项目采用模块化开发思想，按功能逻辑拆分文件，结构清晰、便于维护与扩展，各模块职责明确、协同工作。

##模块说明
#主程序模块（main.c）
程序唯一入口，提供系统主交互界面，统一调度管理员登录、用户登录、打卡签到三大核心系统，并控制文件数据的读取与写入时机，保证程序运行流程稳定。

#用户信息模块（userInfo.c/userInfo.h）
定义会员链表核心结构体，实现会员数据的新增、删除等基础操作，是整个系统的数据存储核心；链表遍历功能整合至管理员界面模块，优化头文件依赖关系。

#打卡与用户业务模块（yxl_daka.c/yxl_user_use.c/yxl.h）
打卡系统负责会员签到验证与记录；用户登录系统支持个人信息查询、手机号修改、卡号状态管理等自助操作，为普通会员提供便捷的服务入口。

#时间与充值模块（rest.c/rest.h）
实现会员卡时间管理功能，包括开卡时间与过期时间展示、时间卡剩余时长计算、会员卡天数充值等功能，支撑时间类会员卡的全生命周期管理。

#管理员系统模块（admin_ui.c/admin_biz.c/admin_list.c/admin.h）
采用分层设计：UI 层负责界面展示、业务层处理核心管理逻辑、链表层维护数据结构；支持会员信息增删改查、数据查看、系统配置等管理员操作，权限明确、操作便捷。

#文件数据管理模块（IO.c/ IO.h）
负责系统数据的持久化存储，程序启动时自动加载用户数据与管理员数据，退出时自动保存所有修改，保证数据不丢失；支持用户数据文件（user_data.txt）与管理员数据文件（admin_data.txt）的读写操作。

##数据结构设计
系统采用单向链表存储会员信息，结构稳定、便于动态增删数据

##核心功能
双角色登录：管理员权限 + 普通用户权限分离管理
会员管理：开卡、信息修改、卡号冻结、数据查询
打卡系统：支持次数卡与时间卡两种模式签到
充值服务：时间卡天数充值、有效期自动更新
时间管理：开卡时间、过期时间查询，剩余时长统计
数据持久化：文件自动读写，断电不丢失数据
可视化控制台菜单：操作简单、流程直观、易于使用

##项目特点
模块化设计，代码结构清晰，便于二次开发与功能扩展
支持两种会员卡类型，满足不同健身人群需求
数据全程内存管理 + 文件存储，安全稳定
菜单式交互界面，无需专业知识即可快速上手
功能完整，覆盖健身房会员管理全流程
